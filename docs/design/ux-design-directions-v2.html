Briefing card — Subtle warm-on-narrative-hero

✓ Stakeholder chosen

Direction 1 · Train tab home view

Weekly Plan — Heti Brief style

✓ Stakeholder chosen

The Train tab home view (when no workout is in progress) shows the upcoming week as a single integrated plan. Narrative paragraph in companion voice opens the week's framing (energy state, mesocycle position, Reta-cycle context), then a vertical day list with mini icon-sets per day shows workout / sport / supplement / meal commitments. Long-tap a row → inline alternative swap dropdown. Approve "Ez lesz a hetem" CTA at bottom.

Rationale Smallest Phase 2-3 → Phase 4-5 transition cost. Phase 2-3 ships the same shell with manual `MesocycleWeek` entry; Phase 4-5 swaps in AI-proposed narrative. The mesocycle context already lives on Train tab, so the planning UX inherits its data-home naturally. Swap-pattern engagement metric (1-2 swaps = healthy per Mary R5 + John R5 + Notion phantom-acceptance precedent) is preserved.

Mockup shows: Sunday evening 21:45, companion has just generated next week's proposal. Daniel can swap rows inline (Tuesday's pull-day has alternative chip), then approve. Swap count visible at bottom (0 swaps = phantom acceptance flag candidate).

Direction 5 · Train workout flow

PR celebration — 60-70% overlay-card

✓ Stakeholder chosen

Sally R8 corrected the original Hevy-toast pattern to a persistent overlay-card — large enough to honor the moment, NOT full-screen (which would break workout flow continuity). The card occupies ~65% of viewport with backdrop dim; Daniel dismisses with tap-outside or "Folytatjuk →" button. Copy is memory-anchored, not stat-toast — references past effort, prior mesocycle position, and contextualizes the PR.

Rationale Step-4 Principle #10 ("never rushes a moment that matters") + Principle #11 ("co-authorship") + Hevy-trigger-logic adoption + Mezo container correction (Sally R8). Full-screen modal would overshadow; bottom-sheet would under-emphasize. The 60-70% card is the visual middle ground that says "stop, this matters" without breaking the set-flow rhythm.

Backdrop-blurred workout flow visible behind the card preserves spatial context. Stat is large + monospace + brand-glow; narrative is companion-voice first-person plural with explicit memory anchor.

Direction 4 · Me → Knowledge Graph

Knowledge Graph — flat-list default + graph toggle

✓ Stakeholder chosen

Me → Knowledge Graph opens as a category-grouped flat fact-card list (v1 mockup style retained) with a view toggle at the top-right to reveal force-directed graph mode. Accessibility-friendly, performant on low fact-count Phase 1-4, understandable without UX-onboarding. The graph mode preserves the brand-moat surface (Step-3 Opportunity #1) for when fact-count grows Phase 5+ (~50+ facts).

Rationale Step-1c open question F.5 explicit recommendation. Phase 1-4 typically has 5-30 facts — a graph would feel sparse and visually under-utilized. The flat list with category color-coding (physiology=brand-glow, trigger=warning, preference=brand-glow lighter, response=success) communicates the L3 memory state immediately. Toggle to graph mode is a 1-tap reveal. Phase 5+ may auto-switch the default once fact-count crosses a threshold.

Notice the toggle indicator at top — "List | Graph" segmented control with "List" active. Filter row below for category filtering. Each fact card shows category, confidence, reinforcement count, and inline em-highlight for the key claim.

Direction 3 · Global QuickInputSheet

QuickInputSheet — voice-first primary

✓ Stakeholder chosen

When Daniel taps the global FAB (visible in every screen above), the QuickInputSheet opens with a single large voice-mic button as the primary modality. Other modalities (photo, choice-chips, number-tap, text) are accessible via a secondary icon row below, but visually deprioritized. IDENT-4 most literal interpretation: voice is the assumed path; everything else is fallback.

Rationale Daniel explicitly says "logging is hard" — the friction wall in every nutrition tracker, every workout app, is the form. Voice removes it. The companion AI parses the voice clip into the right entity (Meal / Niggle / PersonEntry / FreeNoteEntry / WorkoutNote) automatically. Other modalities visible-but-secondary preserves optionality without competing for primary attention. AI-routing (option C) is over-engineered before voice-first Phase 1 validation.

Notice: brand-primary radial-gradient mic button with notched-chamfer 24px and brand-glow ring (recording-state indicator). Hint text below ("Mondd el, mi történt") frames voice as conversation, not command. 5 secondary modalities below ranked by next-most-likely-use: Photo → Chip → Number → Text → Cancel.
