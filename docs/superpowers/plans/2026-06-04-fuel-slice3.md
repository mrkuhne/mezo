**Goal:** Build the **Fuel** tab ‚Äî five sub-views (Mai ¬∑ Terv ¬∑ Stack ¬∑ Receptek ¬∑ Kamra) reached through a sub-navigation, with **all interactive sheets fully functional** (MealScore, Replan, RecipeDetail, NewRecipe + IngredientPicker, IngredientDetail, ImportItem, StackPicker, GymSchedule) ‚Äî to pixel-parity with the locked prototype, on typed mock data.

**Architecture:** New feature slice under `src/features/fuel/`. Mirrors the Slice-2 (Me) pattern: the `fuel` route becomes a **parent route** rendering shared chrome (`FuelSubNav` + `<Outlet/>`) with **one nested child route per sub-view** (`/fuel` index = Mai, `/fuel/plan`, `/fuel/stack`, `/fuel/recipes`, `/fuel/kamra`). A new Fuel mock-data layer (`src/data/fuel.ts`, `src/data/pantry.ts`, `src/data/fuelWeek.ts` + types + read-only hooks) feeds the views. Sheets are **local component state** (not routes) and reuse the existing `Sheet` primitive. Interactive sheets that "save" (GymSchedule, StackPicker, NewRecipe, ImportItem, Replan) mutate **local React state** ‚Äî never a backend, never `window.MezoData` (the prototype mutates globals; we don't).

**IMPORTANT ‚Äî prototype is the source of truth (overrides `02-screens.md`):** `02-screens.md` says the daily fuel-timeline lives under **Terv**. The actual prototype renders it under **Mai** (`fuel.jsx` ‚Üí `FuelScreen` renders `FuelTimeline`), and **Terv** (`fuel-plan.jsx` ‚Üí `FuelPlanView`) is the **weekly** rhythm view (weekly stats ¬∑ 7-day Reta strip ¬∑ 24h rhythm grid ¬∑ recurring patterns ¬∑ weekly supplement matrix ¬∑ gym-schedule editor). Pixel-parity is the acceptance test, so **follow the prototype**: Mai = daily macro hero + daily timeline; Terv = weekly rhythm. Likewise, the "Stack" sub-tab maps to `fuel-stack.jsx` ‚Üí **FuelStackView** (AI protocol builder), per `02-screens.md`. The prototype's separate `FuelStashView` ("Polc" supplement shelf) is **not in the 5-tab nav** and is superseded by Kamra ‚Äî do **not** build it.

export function ProgressBar({ value, tone = 'glow', color, glow, className }:
  { value: number; tone?: Tone; color?: string; glow?: boolean; className?: string }) {
  const pct = Math.max(0, Math.min(100, value))

    <div className={cn('bar', className)}>
      <div className={cn('bar-fill', !color && tone)}
        style={{ width: `${pct}%`, ...(color ? { background: color } : {}),
          ...(glow && color ? { boxShadow: `0 0 6px ${color}` } : {}) }} />
    </div>
  )
}

```tsx
export function ScoreRing({ pct, size = 56, stroke = 4, color = 'var(--brand-glow)', label }:
  { pct: number; size?: number; stroke?: number; color?: string; label?: string }) {
  const r = size / 2 - stroke
  const c = 2 * Math.PI * r
  const offset = c - Math.max(0, Math.min(1, pct)) * c
  return (
    <div style={{ position: 'relative', width: size, height: size }}>
      <svg width={size} height={size} style={{ transform: 'rotate(-90deg)' }}>
        <circle cx={size / 2} cy={size / 2} r={r} fill="none" stroke="var(--surface-2)" strokeWidth={stroke} />
        <circle cx={size / 2} cy={size / 2} r={r} fill="none" stroke={color} strokeWidth={stroke}
          strokeLinecap="round" strokeDasharray={c} strokeDashoffset={offset}
          style={{ transition: 'stroke-dashoffset 0.6s ease', filter: `drop-shadow(0 0 4px ${color})` }} />
      </svg>
      {label != null && (
        <span style={{ position: 'absolute', inset: 0, display: 'flex', alignItems: 'center',
          justifyContent: 'center', fontFamily: 'var(--ff-display)', fontSize: size * 0.28,
          color: 'var(--text-primary)' }}>{label}</span>
      )}
    </div>
  )
}
```
